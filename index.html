I actively develop, host and maintain www.athleticprospects.com for a client. The site includes a MySQL database, with a PHP PDO backend. It also includes custom built user authentication, with secure storage of hashed/salted passwords. The client collects and maintains around 50+ pieces of information from users in order to manage his business. There is also a custom built administration panel for the client to administer the site, by deleting offending user accounts, downloading a copy of the database, as well as posting to a custom built blog. The site also uses PHPMailer to send emails based on events, such as a user requesting to change their password, or a profile being reported.

As the site is in constant development, there are always improvements to be made. There are places where, for expediency I chose to use mysqli to procedurally connect to the database, rather than use OOP. As no frameworks were used to create the site, figuring how the best way to structure the project has been a challenge, and the constant need to manually test site functionality to ensure there weren't any regressions has made clear to me the need to implement unit testing, and later, integration testing.
